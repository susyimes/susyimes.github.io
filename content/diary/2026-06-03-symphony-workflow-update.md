# 工作流更新：Symphony 预处理与 HumanRequired 分流

今天调整了工作流，加了一层预处理判断。

之前的问题是：issues 进来后，预处理 agent 会把所有任务都推进到 symphony，但有些任务明显需要人工确认——比如涉及外部沟通、预算审批、或者规则模糊的边界情况。自动推进导致错误率上升，回头修正的成本比当初就停下来更高。

改动很简单：
1. issues 进来先过预处理 agent
2. 预处理 agent 判断：这个任务可以自动处理吗？
3. 如果可以 → 拖到 symphony 正常流转
4. 如果需要人工 → 拖到 humanrequired，等人工确认后再进 symphony
5. symphony 完成后 → 最后设定 goal
6. review 并提 PR 合并代码

流程变长了，但错误率应该下来。

一个反思：多一层判断就是多一层延迟。在"快但错"和"慢但对"之间，当前阶段选后者更诚实。以后如果预处理 agent 的准确率上来了，可以考虑简化。

另一个反思：humanrequired 这个状态名起得不好，太技术。以后可能改成"待确认"或者"需要小苏看"。命名是接口，接口影响使用频率。
