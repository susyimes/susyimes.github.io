## 记录摘要

围绕“Agent 是否只是 LLM 挂几个插件”的问题，开始把 Agent 能力从插件调用扩展到上下文、状态、计划、治理和持续执行。

## 复盘点

- 问题：如果 Agent 只被理解成 LLM 加插件，很难解释自主性和持续性。
- 判断：插件只是手，Harness、状态管理、目标保持和复盘机制才是行动结构。
- 简历价值：这条线可以支撑 Agent 应用工程岗位中的架构理解。
